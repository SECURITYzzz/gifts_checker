# Gifts Checker

> [English version](README.md)

Telegram-юзербот, который следит за каталогом Star Gifts и публикует сообщение в канал, когда появляется новый подарок или меняется его цена апгрейда. Каждый отслеживаемый подарок зеркалится в личный стикерсет кастомных эмодзи, поэтому в отчёте сразу видно стикер подарка.

## Возможности

- Опрос `get_available_gifts` с настраиваемым интервалом
- Детекция новых подарков и изменения цены апгрейда, один дайджест на итерацию
- Автосоздание стикерсета кастомных эмодзи; `custom_emoji_id` переиспользуется между запусками
- Хранение состояния в SQLite — рестарт не приводит к повторной рассылке
- Периодический статус-отчёт с uptime и success rate
- Автоперезапуск сессии при падении
- Секреты загружаются из `.env`

## Структура проекта

```
├── main.py            # точка входа, логирование, жизненный цикл клиента
├── gift_checker.py    # цикл мониторинга, детекция изменений, статус-отчёт
├── emoji_creator.py   # создание стикерсета и извлечение custom emoji
├── database.py        # обёртка aiosqlite над таблицей gifts
├── config.py          # загрузка настроек из .env
├── .env.example       # шаблон переменных окружения
└── requirements.txt
```

## Установка

```bash
pip install -r requirements.txt
cp .env.example .env
```

Заполнить `.env` своими данными Telegram API и id каналов.

## Запуск

```bash
python main.py
```

При первом запуске Pyrogram запросит код подтверждения для создания файла сессии.

## Настройка

| Переменная | Описание |
|---|---|
| `API_ID`, `API_HASH` | Данные Telegram API с [my.telegram.org](https://my.telegram.org) |
| `PHONE` | Номер в международном формате (например `+123456789`) |
| `CHANNEL_ID` | Чат для публикации изменений (`@username` или числовой id) |
| `STATUS_CHAT_ID` | Чат для статус-отчётов (по умолчанию `me`) |
| `SESSION_NAME` | Имя файла сессии Pyrogram (без расширения) |
| `DB_FILE_PATH` | Путь к файлу SQLite |
| `STICKERSET_TITLE` | Название стикерсета |
| `STICKERSET_SHORT_NAME` | Уникальный short name стикерсета (должен быть глобально уникален) |
| `CHECK_INTERVAL` | Секунды между снимками каталога (по умолчанию `30`) |
| `STATUS_INTERVAL` | Секунды между статус-отчётами (по умолчанию `3600`) |

## Как это работает

1. При старте чекер загружает ранее виденные подарки из SQLite и ищет свой стикерсет кастомных эмодзи (если его нет — создаст при первом новом подарке).
2. Каждые `CHECK_INTERVAL` секунд монитор получает актуальный каталог. Новые подарки добавляются в стикерсет и сохраняются; существующие сравниваются по `upgrade_price`.
3. Если в итерации есть изменения, в `CHANNEL_ID` уходит один HTML-пост: подарок рендерится как custom emoji рядом с разницей цен.
4. Каждые `STATUS_INTERVAL` секунд в `STATUS_CHAT_ID` отправляется короткий отчёт о успешности запросов.
